Photoshop混合模式
2020-12-21 # Photoshop # Notes

混合模式

基色：下方被混合的颜色，基色是原始图像颜色，用字母“J”代替。
混合色：产生混合模式的上层颜色，混合色是通过绘画或编辑工具应用的颜色，用字母“H”代替。
结果色 = 混合色+基色，结果色是混合后得到的颜色。

亮度和明度是两个色彩模式中的不同通道。亮度（brightness）是HSB模式中的通道，同时也是你用RGB模式的时候的三通道的总和也可以叫亮度，明度（lightness）是LAB模式中的通道。

基础型

正常 Normal：只能通过不透明度或填充度混合。当“不透明度”为100%，也就是说完全不透明时，“结果色”的像素将完全由所用的“混 合色”代替；当“不透明度”小于100%时，混合色的像素会透过所用的颜色显示出来，显示的程度取决于不透明度的设置与“基色”的颜色。
溶解 Dissolve：改变透明度或填充度，像素点化的扩散，结果色由基色或混合色的像素随机替换。

在调整不透明度后，图层的图层样式，其颜色跟着图层的不透明度变化，而调整填充后，变化的仅仅是图层本身，图层样式不受影响。填充针对的是对象本身，而不同明度针对的是整个图层。

去亮型（变暗组）

红、绿、蓝三个颜色通道每种色各分为255阶亮度，0表示亮度为0%，255表示亮度为100%，这是我们首先要知道的，数值表示的是颜色的亮度。

变暗 Darken：C = min(H,J)，像素点基于通道亮度比较，谁暗就显示谁。在该模式下，对混合的两个图层相对应区域RGB通道中的颜色亮度值进行比较，在混合图层中，比基色图层暗的像素保留，亮的像素用基色图层中暗的像素替换，会产生新的颜色。

正片叠底 Multiply：（H*J）/255，转换公式：c = h*j。由于h和j都是在0到1的区间取值，所以c肯定小于等于1。白色与任何颜色混合，白色消失。黑色与任何颜色混合，只留黑色。基于通道将上下两层图层像素颜色的灰度级进行乘法计算，获得灰度级更低的颜色而成为合成后的颜色，图层合成后的效果简单地说是低灰阶的像素显现而高灰阶不显现(即深色出现，浅色不出现，黑色灰度级为0，白色灰度级为255)。

图片自己“正片叠底”自己，因为h=j，所以肯定会得，c=h²或者c=j²，无论哪种，这就是一个非常简单的二次函数。和曲线压暗的效果一样，但是正片叠底的压暗是一个固定定值；而曲线压暗是变化值，我们是可以调的。

与纯色图层进行正片叠底，就是简单的调色。高光区变化很快，而阴影区变化的很慢。

颜色加深 Color Burn：C = (H+J-255)*255/H，转换公式：c = 1-(1-j)/h两通道亮度之和低于255，直接归零；两通道中有一个为0，结果直接归零。混合后，暗部会变更暗，对比加强。使用这种模式时，会加暗图层的颜色值，加上的颜色越亮，效果越细腻。让底层的颜色变暗，有点类似于正片叠底，但不同的是，它会根据叠加的像素颜色相应增加对比度。和白色混合没有效果。

如果在上方图层，填充的是100%的纯色时，最后混合出来的图片，是比较容易发生失真现象的。（如果你喜欢这种失真的话，确实可以玩玩。）

如果纯色层在上层，正常照片层在下层：在发生颜色加深时，输出的函数是从阴影处向高光处逼近的，但是下层100%的白色区域不会被波及。

通过增加阴影部分来增加对比度。

线性加深 Linear Burn：C = H+J-255，转换公式：c = h+j-1。两通道亮度之和低于255，直接归零；两通道中有一个为0，结果直接归零，与颜色加深类似但画面对比没有颜色加深那么强烈。

当B层为纯色层时，那么点B就相当于一个常数b,所以C=A+b-1了。我们切换全局视角，看看b取不同值的时候，究竟会怎样？

我们来观察一下线性加深 的函数图像，真的非常线性，无论是自己混合自己，还是和纯色层混合，就从来没有 弯过。这下应该就懂得了，为什么叫做线性加深了吧。

原本亮的地方变暗了，原本暗的地方变更暗，甚至是黑了。整体变暗。

深色 Darker Color：R+G+B:H>J保留J部分反之保留H部分，保留暗部。不是混合色就是基色，不会变色。深色与变暗的区别：变暗是基于像素点RGB三个通道分别进行比较，保留暗的通道，会产生新颜色。深深色是把一个像素的三个通道的值加起来，然后和另一个图层的像素三个通道加起来做比较。然后数值小的像素留下，不会产生新颜色。

去暗型（变亮组）

变亮 Lighten：C = Max(H,J)。与变暗模式相反，像素点基于通道亮度比较，谁亮就显示谁。

滤色 Screen：255-{(255-H)*(255-J)}/255，变换公式：c = 1-(1-h)*(1-j)。白色与任何色混合只留白色，黑色和任何色混合，黑色消失，画面整体变亮。高光地区缓慢变亮，阴影部分急速变亮。越是黑暗的地方被消灭的越快。

颜色减淡 Color Dodge：J+{(H*J)/(255-H)}，混合后，亮部会变更亮，对比加强。越是光明的地方变亮的变化迅速，阴影处变化缓慢。

**线性减淡 Linear Dodge (Add)**：C = H+J，滤色和颜色减淡的综合，最大范围变亮。

浅色 Lighter Color：R+G+B:H>J，保留H部分反之保留J部分，保留亮部。不是混合色就是基色，不会变色。

叠加组（对比组）

叠加 Overlay：根据基色判断。如果基色>128，结果色采用滤色算法（半值）255-{(255-H)*(255-J)}/128；如果基色<128，结果色采用正片叠底算法（半值）（混合色*基色）/128；如果基色=128，结果色=基色。以底层来决定上层是否执行两倍的正片叠底或者两倍的滤色的。说两倍正片叠底，意思不是说比原来的更加黑了，而是比原来更加淡了。

当上层B层大于50%灰的时候，图像变亮了。当上层B层小于50%灰的时候，图像变暗了。2倍的正片叠底当然可以让底层变亮呀。举个例子，顶层是80%的灰，底层是40%灰。如果执行是普通的正片叠底，就是C=AB=80%40%=32%，结果层变暗了。如果执行的是叠加，由于底层是小于50%灰，所以执行2倍的正片叠底，就是 C=2AB=2*40%*×80%=64%，结果层变亮了。

柔光 Soft Light：根据混合色判断。如果混合色>128，变亮；如果混合色<128，变暗，程度较低比较柔和。如果混合色=128（中性灰），结果色=基色。同样，这个模式和叠加一样，当B层大于50%灰的时候，结果层是变亮的。当B层小于50%灰的时候，结果层变暗了。可以看到无论是变亮还是变暗，图像的阴影区域变化 都要比高光要来的猛烈。

强光 Hard Light：算法与叠加一样，但根据混合色判断。如果混合色>128，结果色采用滤色算法（半值）255-{(255-H)*(255-J)}/128；如果混合色<128，结果色采用正片叠底算法（半值）（混合色*基色）/128。如果混合色为白色，结果色=白色；如果结果色为黑色，结果色=黑色。

当混合层 H自己与自己强光模式叠加，效果与叠加模式一样。

那么整个H层 从灰到黑，从黑到白时，结果层C 是怎么变化的呢？

亮光 Vivid Light：根据混合色判断。如果混合色>128，类似“颜色减淡”；如果混合色<128，类似“颜色加深”，强度更弱；与柔光近似，但比柔光更强烈。

线性光 Linear Light：2*H+255-J，H=128（中性灰），结果为基色；H>128，变亮，超过255都视为255；H<128，变暗，出现负数都视为0。

点光模式 Pin Light：高于128的混合色替换比它暗的基色；低于128的混合色替换比它亮的基色；等于128的混合色是中性色，无替换效果。白色或黑色（255/0）会替换所有基色。

实色混合 Hard Mix：混合色的RGB值添加到基色的RGB值。如果结果总和>=255，则值为255；如果值<=255，则为0。因此，此模式结果只会有8中基本色，及RGBCMY和黑白。

色差型

差值 Difference：查看每个通道中颜色信息，并从基色中减去混合色或从混合色中减去基色，具体取决于哪一个颜色的亮度值更大。

排除 Exclusion：创建一种与“差值”模式相似，但对比度更低的效果。排除就是弱化版的差值。调色：高光是阴影的互补色。

减去 Subtract：查看每个通道中的颜色信息，并从基色中减去混合色，负值为零。混合色月亮，结果色越暗。混合色越暗，结果色变暗的程度越低。中性色为黑色。

划分 Divide：(J/H)*255，基色大于等于混合色，结果为白色。基色小于混合色，结果色加深基色。混合色为白色，结果色为基色。混合色为黑色，结果色为白色。中性色是白色。

调色型

色相 Hue：用基色的明亮度和饱和度以及混合色的色相来创建结果色。

饱和度 Saturation：用基色的明亮度和色相以及混合色的饱和度来创建结果色。

颜色 Color：用基色的明亮度以及混合色的色相和饱和度来创建结果色。黑白图像用来上色。

明度 Luminosity：用基色的色相和饱和度以及混合色的明亮度来创建结果色。

穿透：组的“正常模式”，到组的最后一次结束，不与组下的图层进行混合。穿透模式可以穿过最后一层，与组下面的图层进行混合。

颜色混合带合成

本图层的调整就是本图层颜色的移除，露出下面的图层。下一图层调整就是使下一图像上浮，覆盖掉本图层。
